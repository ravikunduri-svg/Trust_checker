 HOW IT WORKS (AUTOMATIC)
═══════════════════════════════════════════════════════════════════════════

STEP 1: CUSTOMER RUNS COMMAND
  dseries_healthcheck.bat C:\CA\ESPdSeriesWAServer_R12_4

STEP 2: SCRIPT AUTO-DETECTS JAVA
  ✅ Tries: C:\CA\ESPdSeriesWAServer_R12_4\jre\bin\java.exe (dSeries bundled)
  ✅ Falls back to: System Java from PATH or JAVA_HOME
  ✅ Result: Java found and verified

STEP 3: SCRIPT AUTO-BUILDS CLASSPATH
  ✅ Adds: dseries-healthcheck.jar
  ✅ Adds: C:\CA\ESPdSeriesWAServer_R12_4\lib\*.jar (JDBC drivers)
  ✅ Adds: C:\CA\ESPdSeriesWAServer_R12_4\third-party\*.jar
  ✅ Adds: C:\CA\ESPdSeriesWAServer_R12_4\ext\*.jar
  ✅ Result: Complete classpath with all dependencies

STEP 4: TOOL READS DSERIES CONFIG
  ✅ Reads: C:\CA\ESPdSeriesWAServer_R12_4\conf\db.properties
  ✅ Parses: JDBC URL, username, password
  ✅ Decrypts: Password if encrypted (ENC format)
  ✅ Result: Database connection details ready

STEP 5: TOOL RUNS HEALTH CHECKS
  ✅ System checks (CPU, memory, disk)
  ✅ Server checks (JVM heap, ports, logging)
  ✅ Database checks (table sizes, integrity, performance)
  ✅ Agent checks (connectivity, versions)
  ✅ Client checks (connections, failures)
  ✅ HA checks (mode, nodes, failovers)
  ✅ Result: Comprehensive health assessment

STEP 6: TOOL GENERATES REPORT
  ✅ Creates: health_check_report_YYYYMMDD_HHMMSS.txt
  ✅ Includes: All check results, health score, remediation
  ✅ Displays: Summary on screen
  ✅ Returns: Exit code (0=success, 1=failure, 2=error)

CUSTOMER EXPERIENCE:
  One command → Complete health check → Detailed report
  Time: 2-5 minutes
  Configuration: NONE required!

 TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════════════

JAVA DETECTION LOGIC:

  Priority 1: dSeries bundled Java
    - Windows: <install_dir>\jre\bin\java.exe
    - Unix: <install_dir>/jre/bin/java

  Priority 2: System Java
    - JAVA_HOME environment variable
    - Java in PATH

  If not found:
    - Clear error message
    - Instructions to install Java
    - Exit code 2

CLASSPATH BUILDING:

  Base: dseries-healthcheck.jar

  Auto-added:
    - <install_dir>/lib/*.jar (includes PostgreSQL JDBC driver)
    - <install_dir>/third-party/*.jar
    - <install_dir>/ext/*.jar
    - AIX: <install_dir>/aix_lib/*.jar (if on AIX)

  Result: Complete classpath with all JDBC drivers and dSeries libraries

DATABASE CONFIGURATION:

  Source: <install_dir>/conf/db.properties (dSeries config)

  Reads:
    - jdbc.URL (connection string)
    - rdbms.userid (username)
    - rdbms.password (encrypted or plain)
    - rdbms.driver (JDBC driver class)

  Password Decryption:
    - Plain text: Used directly
    - ENC(base64): Decoded automatically
    - {ENCRYPTED}: Requires dSeries encryption library (graceful fallback)

  Security:
    - All credentials masked in output
    - No credentials stored by tool
    - Read-only database access

OS DETECTION:

  Detected automatically:
    - Windows (all versions)
    - Linux (all distributions)
    - AIX
    - Solaris
    - HP-UX
    - MacOS

  OS-specific handling:
    - Path separators (\ vs /)
    - Classpath separator (; vs :)
    - Java options
    - Library paths
