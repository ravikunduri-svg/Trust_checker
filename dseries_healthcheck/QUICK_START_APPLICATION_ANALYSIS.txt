┌───────────────────────────────────────────────────────────────────────────────┐
│ QUICK START (3 STEPS)                                                         │
└───────────────────────────────────────────────────────────────────────────────┘

  1. PREPARE: Place application XMLs in apps/ directory
     
     Windows:   C:\CA\ESPdSeriesWAServer_R12_4\apps\*.xml
     Unix:      /opt/CA/ESPdSeriesWAServer_R12_4/apps/*.xml

  2. RUN: Execute health check as normal
     
     Windows:   dseries_healthcheck.bat C:\CA\ESPdSeriesWAServer_R12_4
     Unix:      ./dseries_healthcheck.sh /opt/CA/ESPdSeriesWAServer_R12_4

  3. REVIEW: Check "APPLICATION BEST PRACTICES ANALYSIS" section

┌───────────────────────────────────────────────────────────────────────────────┐
│ COMMON FIXES                                                                  │
└───────────────────────────────────────────────────────────────────────────────┘

  1. HARDCODED CREDENTIALS
     ─────────────────────
     ❌ BAD:  <command>mysql -u admin -p MyPassword123 mydb</command>
     ✅ GOOD: <command>mysql -u %VAR('DB_USER') -p %VAR('DB_PASSWORD') mydb</command>

  2. HARDCODED PATHS
     ───────────────
     ❌ BAD:  <command>C:\data\scripts\process.bat</command>
     ✅ GOOD: <command>%VAR('SCRIPT_DIR')\process.bat</command>

  3. CLOUD MIGRATION (AWS S3 Example)
     ─────────────────────────────────
     ❌ OLD:  <cmd_job> with "aws s3 cp file.txt s3://bucket/"
     ✅ NEW:  <pojo_job> with S3Pojo.upload() method

│ TROUBLESHOOTING                                                               │
└───────────────────────────────────────────────────────────────────────────────┘

  Q: No applications found?
  A: • Check XMLs are in apps/ directory
     • Verify file permissions
     • Ensure valid XML syntax

  Q: Missing cloud opportunities?
  A: • Tool detects CLI commands in <command> tags
     • Check for standard CLI syntax (aws, az, gcloud, etc.)
     • Custom wrapper scripts may not be detected

  Q: How to implement recommendations?
  A: • See APPLICATION_BEST_PRACTICES_GUIDE.md for detailed examples
     • Follow links to Broadcom plugin documentation
     • Test changes in development environment first
