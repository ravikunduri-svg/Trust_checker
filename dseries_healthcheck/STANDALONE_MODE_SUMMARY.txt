┌───────────────────────────────────────────────────────────────────────────────┐
│ QUICK START                                                                   │
└───────────────────────────────────────────────────────────────────────────────┘

  STEP 1: Export Applications
  ────────────────────────────
  Desktop Client:  Right-click → Export → Save as XML
  CLI:             espappexport -A APP_NAME -f output.xml
  REST API:        GET /application/{name}/export

  STEP 2: Run Analysis
  ─────────────────────
  Windows:         dseries_healthcheck.bat --analyze-apps C:\exports\
                   dseries_healthcheck.bat -a C:\exports\app.xml

  Linux/Unix:      ./dseries_healthcheck.sh --analyze-apps /exports/
                   ./dseries_healthcheck.sh -a /exports/app.xml

  Direct Java:     java -jar dseries-healthcheck.jar -a /path/to/apps

  STEP 3: Review Results
  ──────────────────────
  • Analysis summary
  • Best practice violations (by severity)
  • Cloud integration opportunities
  • Specific recommendations

┌───────────────────────────────────────────────────────────────────────────────┐
│ WHAT GETS ANALYZED                                                            │
└───────────────────────────────────────────────────────────────────────────────┘

  SECURITY (HIGH Priority)
  ────────────────────────
  ⚠️  Hardcoded credentials in commands
  ⚠️  Hardcoded credentials in POJO parameters
  ⚠️  Password exposure in XML files

  CONFIGURATION (MEDIUM Priority)
  ────────────────────────────────
  ℹ️  Hardcoded file system paths
  ℹ️  Large inline JavaScript/scripts
  ℹ️  Missing application defaults

  RELIABILITY (LOW Priority)
  ──────────────────────────
  💡 Missing retry configuration
  💡 No calendar usage

  CLOUD OPPORTUNITIES
  ───────────────────
  ☁️  AWS CLI → AWS plugins (S3, Lambda, Batch, Glue, EMR, etc.)
  ☁️  Azure CLI → Azure plugins (Blob, Data Factory, Synapse, etc.)
  ☁️  GCP CLI → GCP plugins (Storage, Dataproc, BigQuery, etc.)
  ☁️  Databricks CLI → Databricks plugin
  ☁️  FTP jobs → Cloud storage plugins

┌───────────────────────────────────────────────────────────────────────────────┐
│ EXAMPLES                                                                      │
└───────────────────────────────────────────────────────────────────────────────┘

  # Analyze directory of exported applications
  dseries_healthcheck.bat --analyze-apps C:\exports\applications

  # Analyze single application file
  dseries_healthcheck.bat -a C:\exports\PAYROLL_APP.xml

  # Linux/Unix
  ./dseries_healthcheck.sh --analyze-apps /home/user/exports

  # Analyze and save report
  dseries_healthcheck.bat -a C:\exports > analysis_report.txt

  # CI/CD pipeline integration
  java -jar dseries-healthcheck.jar -a ./apps/ || exit 1

┌───────────────────────────────────────────────────────────────────────────────┐
│ EXPORT METHODS                                                                │
└───────────────────────────────────────────────────────────────────────────────┘

  DESKTOP CLIENT (GUI)
  ────────────────────
  1. Right-click on Application
  2. Select "Export" → "Export Application"
  3. Save as XML file

  COMMAND LINE (CLI)
  ──────────────────
  # Single application
  espappexport -A PAYROLL_PROCESSING -f PAYROLL.xml

  # Multiple applications (pattern)
  espappexport -A "PROD_*" -f prod_apps.xml

  # All applications
  espappexport -A "*" -f all_apps.xml

  REST API
  ────────
  # Using curl
  curl -X GET "http://server:9443/application/APP_NAME/export" \
    -H "Authorization: Bearer TOKEN" -o app.xml

  # Using PowerShell
  Invoke-RestMethod -Uri "http://server:9443/application/APP_NAME/export" `
    -Headers @{Authorization="Bearer TOKEN"} -OutFile "app.xml"

┌───────────────────────────────────────────────────────────────────────────────┐
│ TROUBLESHOOTING                                                               │
└───────────────────────────────────────────────────────────────────────────────┘

  Q: "No XML files found"
  A: • Check file extension is .xml
     • Verify file permissions
     • Ensure correct directory path

  Q: "Parse errors"
  A: • Verify files are valid dSeries exports
     • Check XML syntax
     • Try exporting again

  Q: "Java not found"
  A: • Install Java 8 or later
     • Add Java to PATH
     • Use full path to java command
