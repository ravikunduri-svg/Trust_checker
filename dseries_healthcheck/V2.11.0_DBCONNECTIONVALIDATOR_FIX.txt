dSeries Health Check Tool - Version 2.11.0 Release Summary
===============================================================================
Date: 2026-02-13
Critical Fix: Database Connection Logic Rewritten to Match DBConnectionValidator

===============================================================================
CRITICAL CHANGES
===============================================================================

1. COMPLETE REWRITE OF connectToDatabase() METHOD
   - Now EXACTLY matches DBConnectionValidator.java logic
   - Analyzed official dSeries validateDbConnection.bat and DBConnectionValidator.java
   - Replicated the exact connection flow used by dSeries itself

2. PROPERTY HANDLING (EXACTLY LIKE DBConnectionValidator)
   - Loads ALL properties from db.properties
   - Removes specific properties before passing to driver:
     * jdbc.Driver
     * rdbms.driver
     * jdbc.URL
     * integratedSecurity
     * rdbms.userid (after converting to "user")
     * rdbms.password (after converting to "password")
   - Passes ALL remaining properties to DriverManager.getConnection()
   - This ensures SSL properties, custom driver properties, etc. are all passed

3. PASSWORD DECRYPTION (EXACTLY LIKE DBConnectionValidator)
   - Uses Scrambler.recover(password, key) via reflection
   - Gets key from RelationalDatabaseManager.getKey() dynamically
   - Falls back to "RelationalDatabaseManager" if getKey() not available
   - Decrypts both database password and SSL trustStore password
   - Sets system properties for SSL (required for DB2 and Oracle-SSL)

4. WINDOWS AUTHENTICATION SUPPORT
   - Checks integratedSecurity property
   - Appends to JDBC URL if needed
   - Skips user/password if Windows auth is enabled

5. SSL/TLS SUPPORT
   - Decrypts javax.net.ssl.trustStorePassword
   - Sets system properties for trustStore and trustStorePassword
   - Required for DB2-SSL, Oracle-SSL, PostgreSQL-SSL

===============================================================================
TECHNICAL DETAILS
===============================================================================

OLD APPROACH (v2.10.0 and earlier):
- Loaded config once in loadDatabaseConfig()
- Built Properties object with only user/password
- Added SSL properties separately
- Didn't remove original properties
- Didn't pass all db.properties to driver

NEW APPROACH (v2.11.0):
- Loads db.properties fresh in connectToDatabase()
- Removes specific properties (jdbc.Driver, jdbc.URL, etc.)
- Converts rdbms.userid -> "user", rdbms.password -> "password"
- Removes rdbms.userid and rdbms.password after conversion
- Passes ALL remaining properties to DriverManager
- Sets system properties for SSL (required for some drivers)

===============================================================================
WHY THIS FIXES THE ORA-01017 ERROR
===============================================================================

The previous approach was missing properties that Oracle driver needs:
- Custom Oracle connection properties in db.properties
- SSL certificate properties
- Driver-specific configuration

By passing ALL properties (after removing only the non-driver ones),
the connection now works EXACTLY like dSeries's own validateDbConnection.
